# Claims to Evidence Map

This document maps manuscript claims to reproducible artifacts generated by the canonical pipeline (`python mvp/simulation/reproduce_core.py` for end-to-end; `bash hpc/hpc_run.sh` for the 20-seed benchmark).

> **Reproducibility note.** The artefacts below are produced by the
> 20-seed HPC pipeline. The repository's `mvp/simulation/results/`
> directory is scratch space; re-run the HPC pipeline (or extract the
> latest tagged HPC archive) before consulting any of the listed files.

## Core Claims

| Claim ID | Manuscript claim | Primary evidence artifacts | Validation rule |
|---|---|---|---|
| C1 | AGRI-BRAIN outperforms context baselines across core scenarios on ARI/SLCA and reduces waste/carbon | `mvp/simulation/results/benchmark_summary.json`, `benchmark_significance.json`, `paper_benchmark_table.json` | Mean differences and pair-aware tests per (scenario, baseline, metric) |
| C2 | Gains remain under multi-seed uncertainty | `benchmark_summary.json` | `n=20` seed policy and 95 % percentile-bootstrap CIs present for reported metrics |
| C3 | Statistical significance uses pair-aware tests with multiplicity control | `benchmark_significance.json`, `docs/STATISTICAL_METHODS.md` | Required fields per record: `p_value`, `p_value_adj`, `cohens_dz` (paired) or `cohens_d_pooled` (unpaired), `hedges_g`, `effect_size_ci_low/high`, `mean_diff_ci_low/high`, `is_paired_design`, `test_type`, `correction_method` |
| C4 | Context interoperability (MCP + piRAG) contributes to quality improvements | `benchmark_significance.json`, `mcp_protocol_*.json`, `mcp_interop_*.json`, `context_alignment_*.json` | Compare `agribrain` vs `mcp_only`, `pirag_only`, `no_context`; inspect protocol traces and context-honour rate |
| C5 | Robustness under stress perturbations is bounded and auditable across multiple seeds | `stress_summary.json`, `stress_degradation.csv`, `stress_passfail.csv` | `STRESS_N_SEEDS` (default 5) seeds per cell with Clopper-Pearson 95 % CI on the pass proportion |
