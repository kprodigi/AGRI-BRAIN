# Claims to Evidence Map

This document maps manuscript claims to reproducible artifacts generated by the canonical pipeline (`python mvp/simulation/reproduce_core.py` for end-to-end; `bash hpc/hpc_run.sh` for the 20-seed benchmark).

> **Reproducibility note.** The artefacts below are produced by the
> 20-seed HPC pipeline. The repository's `mvp/simulation/results/`
> directory is scratch space; re-run the HPC pipeline (or extract the
> latest tagged HPC archive) before consulting any of the listed files.

## Core Claims

| Claim ID | Manuscript claim | Primary evidence artifacts | Validation rule |
|---|---|---|---|
| C1 | AGRI-BRAIN outperforms context baselines across core scenarios on ARI/SLCA and reduces waste/carbon | `mvp/simulation/results/benchmark_summary.json`, `benchmark_significance.json`, `paper_benchmark_table.json` | Mean differences and pair-aware tests per (scenario, baseline, metric) |
| C2 | Gains remain under multi-seed uncertainty | `benchmark_summary.json` | `n=20` seed policy and 95 % BCa bootstrap CIs (with percentile fallback when the BCa correction is undefined; `_meta.bca_fallback_stats.fallback_rate` surfaces the fraction of cells that fell back) present for reported metrics |
| C3 | Statistical significance uses paired tests with multiplicity control | `benchmark_significance.json`, `docs/STATISTICAL_METHODS.md` | Required fields per record: `p_value`, `p_value_adj`, `p_value_adj_holm` / `p_value_adj_holm_extended` / `p_value_adj_holm_full` (auxiliary FWER columns), `p_value_adj_bh` / `p_value_adj_by` (FDR columns), `cohens_d_pooled` (canonical), `cohens_dz` (paired alongside), `hedges_g` (small-sample correction), `effect_size_ci_low/high` with `effect_size_ci_method = "BCa"`, `mean_diff_ci_low/high`, `is_paired_design = true` for all five baselines (post-2026-04 pairing-extension), `test_type`, `correction_method` |
| C4 | Context interoperability (MCP + piRAG) contributes to quality improvements | `benchmark_significance.json` (channel-decomposition family + `agribrain_vs_{mcp_only,pirag_only,no_context}`), `mcp_protocol_*.json`, `mcp_interop_*.json`, `context_alignment_*.json` | Direct test: read `mcp_only_vs_no_context` and `pirag_only_vs_no_context` per scenario; both must show positive `mean_diff` and significant `p_value_adj_holm_channel < 0.05` (Holm correction within the family of 10 ARI tests; family-corrected p-values also surfaced as `channel_decomposition_holm_adjusted` at the JSON top level). The complementary `agribrain_vs_{mcp_only,pirag_only}` contrasts test whether *adding* the second channel on top of one improves further (some scenarios will show null effects there — interpret as "the single channel saturates this scenario", not as "the channel is unnecessary"). Inspect protocol traces and context-honor rate to corroborate the channel-active mechanism. |
| C5 | Robustness under stress perturbations is bounded and auditable across multiple seeds | `stress_summary.json`, `stress_degradation.csv`, `stress_passfail.csv` | `STRESS_N_SEEDS` (default 5) seeds per cell with Clopper-Pearson 95 % CI on the pass proportion |
| C6 | Within-trace temporal stability across early/mid/late thirds (NOT external validity) | `external_validity_summary.json`, `external_validity_summary.csv`, `external_validity_deltas.csv` (legacy filenames; produced by `run_external_validity.py`) | Confirm AGRI-BRAIN ranking/gains persist across the three thirds of the same 288-row trace; do not advertise as external validity |
| C7 | Results are reproducible and provenance-pinned | `artifact_manifest.json`, aggregator stdout (logs/) | Manifest recursively includes SHA-256 hashes for `benchmark_seeds/eace598_*/seed_*.json` (per-seed evidence) and `decision_ledger/*.jsonl` (per-step audit trail), plus the exact `git_commit` (refused if the working tree is dirty without explicit `AGRIBRAIN_ALLOW_DIRTY=1`); per-cell bootstrap RNG seeds use `blake2b` so they are stable across `PYTHONHASHSEED` / process / OS boundaries |
| C8 | Figures reflect benchmark uncertainty where applicable | `fig6_cross.*`, `fig7_ablation.*`, `fig8_green_ai.*`, `fig9_robustness.*`, `fig10_latency_quality_frontier.*` | CI bars / uncertainty overlays visible in final rendered plots |
| C9 | §4.7 sensitivity to context priors | `table2_ablation.csv` rows for `agribrain_pert_{10,25,50}` (with-learning) and `agribrain_pert_{10,25,50}_static` (no-learning) | Both flavours of perturbation modes ship in the canonical pipeline |
